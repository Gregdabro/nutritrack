## ✅ Что реализовано правильно

**Спринт 5 (Самочувствие и вес)** закрыт полностью. `routes/wellbeing.js` и `routes/weight.js` реализованы с валидацией Zod, авторизацией, typed errors. В `weight.js` — корректная реализация скользящего среднего с хитрым приёмом: запрашивается `limit + 6` записей, чтобы окно MA было полным для крайних точек. Это правильное инженерное решение. Бот-хэндлеры `weight.js` и `wellbeing.js` работают, причём `weight.js` при сохранении также обновляет `user.weightKg` — это аккуратная синхронизация профиля.

**Спринт 6 (Dashboard и Recipes)** реализован полно. `GET /dashboard/today` возвращает весь контракт из раздела 5.9 ТЗ включая `repeatSuggestion`. `GET /dashboard/week` возвращает `days`, `weeklyFoodCost`, `avgNutrients`, `goalsCompletion` — точно по ТЗ. `dashboardStore.js` в Zustand — чистая реализация с отдельными флагами загрузки для `today` и `week`.

**Спринт 7 (Напоминания)** — ключевые части на месте. `User.js` содержит полную структуру `reminders` с дефолтными значениями. `reminderService.js` корректно использует `Intl.DateTimeFormat` для timezone-aware сравнения времени. Вечернее напоминание проверяет, не зафиксировано ли уже самочувствие — правильное поведение. `settings.js` роут с GET/PUT `/reminders`. Фронтенд Settings реализует обе секции: профиль (вес, рост, таймзона) и напоминания с `<input type="time">`.

**ADR compliance** — все 9 ADR соблюдены без исключений.

## ❌ Что не реализовано (конкретные пункты ТЗ)

**1. Команды `/stats`, `/repeat`, `/help` в боте отсутствуют** (раздел 6.1 ТЗ). В `bot/index.js` зарегистрированы: `/today`, `/train`, `/feel`, `/weight`, `/cancel`, `/login`. Команды `/stats`, `/repeat`, `/help`, `/goals` — нет ни одной из них.

**2. Нет API-эндпоинта для обновления профиля пользователя (вес/рост/таймзона)**. В `settings.js` роуте есть только `/reminders`. Фронтенд это знает — в коде Settings буквально написано `// For now, we assume user profile is updated via some api.auth.update() or similar if it exists` и проверяется `if (api.auth && api.auth.updateProfile)` — которого не существует. Поля вес/рост/таймзона в форме Settings **никуда не сохраняются**.

**3. `goalsStore.js` отсутствует** (раздел 3 ТЗ перечисляет его как обязательный Zustand store). Функциональность Goals работает, но через прямые API-вызовы из компонентов, без store.

**4. `test:nutrition` скрипт отсутствует в `package.json`**. `nutritionCalc.test.js` лежит в `src/tests/` и не подключён ни к какому npm-скрипту. Запустить его можно только зная путь вручную.

## ⚠️ Что реализовано с отклонением от ТЗ

**1. Форма тренировки на вебе не позволяет добавлять упражнения.** `AddWorkoutModal` в `Workouts/index.jsx` позволяет задать только метаданные: название, тип, длительность, сложность, заметки. Поля для упражнений (название + подходы × повторения × вес) отсутствуют. Через Telegram-бот AI парсит упражнения полноценно — паритет нарушен. Это самый значимый функциональный gap.

**2. `waterMl` в dashboard возвращает всегда `0`** с комментарием `// TODO: waterMl — учёт воды не реализован в MVP`. ТЗ раздел 5.9 включает `waterMl` в контракт `/dashboard/today`. Вода как отдельный модуль не реализована, хотя цель по воде хранится в `Goal`.

**3. `cron.schedule('* * * * *', ...)` — проверка каждую минуту** при N пользователях делает N запросов к MongoDB каждую минуту. Для двух пользователей некритично, но паттерн уязвим: если по какой-то причине обработка одного цикла займёт больше минуты, следующий запустится параллельно. Правильнее было бы `node-cron` на конкретные минуты (`0,1,2...59 * * * *`) или guard через флаг `isRunning`.

**4. Цвета NutrientBar захардкожены** и не совпадают с CSS-переменными дизайн-системы из `index.css`. Клетчатка отображается фиолетовым (`#9f7aea`) вместо бирюзового (`--nt-teal-mid`).

**5. В Settings нет отдельной кнопки сохранения профиля** — профиль и цели объединены в одну форму с одной кнопкой «Сохранить», хотя `/goals` и (будущий) `/settings/profile` — разные эндпоинты. Сейчас профиль молча не сохраняется.

## 🧹 Наблюдения по коду

В `dashboard.js` суммирование `foodTotals` дублирует логику из `nutritionCalc.js` — та же операция написана вручную циклом `for...of`. Это не критично (два пользователя, не горячий путь), но стоит зафиксировать как технический долг.

В `reminderService.js` вечернее напоминание отправляет клавиатуру с `callback_data: 'feel_great'` и т.д. — эти callback-actions зарегистрированы в `bot/index.js`, так что всё корректно замкнуто. Хорошая деталь.

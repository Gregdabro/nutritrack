# NutriTrack — Полная концепция продукта

> Версия 1.0 | Персональный HealthTech-трекер питания, тренировок и бюджета

---

## 1. Видение продукта

**Одно предложение:** NutriTrack — это персональный цифровой советник по питанию и здоровью, который объединяет учёт еды, тренировок, самочувствия и расходов в единую систему, находит скрытые закономерности и помогает принимать осознанные решения каждый день.

**Расширенное видение:**

Большинство людей знают, *что* нужно делать — правильно питаться и тренироваться. Но они не видят *связей*: почему после определённой еды плохой сон, почему вес не уходит при соблюдении диеты, какой продукт даёт максимум белка за минимум денег. NutriTrack решает именно эту проблему — он не просто считает калории, а думает вместе с пользователем.

**Три кита продукта:**
1. **Простота ввода** — написал боту в Telegram, что съел, и забыл. Остальное система делает сама.
2. **Экономическая осознанность** — первый трекер питания, который показывает не только «сколько съел», но и «сколько заплатил за каждый грамм белка».
3. **Умная аналитика** — система замечает паттерны, которые человек сам не видит.

## 2. Проблемы пользователей

### Проблема 1: Хаос в голове
Пользователь держит план питания в голове. Это работает до первого стресса, усталости или изменения расписания. Нет системы → нет прогресса → нет мотивации.

### Проблема 2: Неизвестная стоимость питания
Человек покупает продукты, тратит деньги, но не понимает: «Я плачу 8€ за 38 г белка в тунце, хотя за 1€ могу получить 60 г белка из курицы». Деньги расходуются, но питание не оптимизируется.

### Проблема 3: Нет связи между данными
Самочувствие плохое — почему? Прогресс остановился — из-за чего? Все данные лежат в разных приложениях или вообще нигде. Никто не соединяет питание → сон → тренировки → вес в единую картину.

### Проблема 4: Сложные приложения
MyFitnessPal, Cronometer — мощные, но перегруженные. Для нетехнического пользователя (второй член семьи) они становятся барьером, а не помощником.

### Проблема 5: Нет контекста «зачем»
Трекеры показывают цифры, но не объясняют их значение. «Тебе не хватило 30 г белка» — и что? NutriTrack должен говорить: «При таком дефиците белка твои мышцы восстанавливаются медленнее после тренировки».

## 3. Ценность продукта

| Для кого | Что получает |
|----------|-------------|
| Технически грамотный пользователь | Полный контроль над питанием, тренировками, бюджетом на еду и аналитикой зависимостей |
| Нетехнический пользователь | Простой Telegram-бот с чёткими инструкциями, понятные ответы, минимум усилий |
| Пара / семья | Общие цели, взаимная поддержка, видимость прогресса друг друга |

**Уникальное конкурентное преимущество:**
- Единственный трекер, считающий **стоимость нутриентов** (€/г белка, €/г углеводов)
- Сравнение цен одного продукта в разных магазинах (Conad vs Lidl vs Pan)
- Система выявления причинно-следственных связей между питанием, тренировками и самочувствием
- Семейный режим с совместными целями и поддержкой

## 4. Пользовательские сценарии

### Сценарий 1: Утренний старт (ежедневный)
> Пользователь просыпается, взвешивается, открывает Telegram.

1. Бот присылает утреннее сообщение: «Доброе утро! Вчера ты выполнил план по белку на 87%. Сегодня до цели: Б 100г / Ж 100г / У 200г / К 30г».
2. После завтрака: пишет «Съел 3 яйца, 150г гречки и кофе».
3. Бот мгновенно отвечает расчётом и обновлённым прогрессом.

### Сценарий 2: Загрузка чека (еженедельный)
> Пользователь вернулся из Conad с покупками.

1. Фотографирует чек прямо в Telegram.
2. Система распознаёт позиции, предлагает сопоставить с базой продуктов.
3. Обновляет цены и запасы продуктов.
4. Показывает: «Куриная грудка в Conad — 5.90€/кг. В прошлый раз в Lidl было 4.80€/кг».

### Сценарий 3: Еженедельная аналитика (воскресенье)
> Пользователь открывает веб-приложение.

1. Видит дашборд за неделю: выполнение плана по БЖУ, потраченный бюджет, граф тренировок, динамику веса.
2. В блоке инсайтов: «Заметили: в дни после тренировки ты потребляешь на 40г меньше белка. Это может замедлять восстановление».
3. Видит сравнительные графики «эта неделя vs прошлая».

### Сценарий 4: Ввод тренировки
> Пользователь закончил занятие с гирей.

Пишет боту: «Сделал: турник 3×10, отжимания 3×20, приседания с гирей 32кг 4×15, длительность 45 минут, сложность 7/10».

Бот рассчитывает калорийный расход, корректирует дневную норму калорий, сохраняет запись.

### Сценарий 5: Оценка самочувствия
> Вечером бот присылает напоминание.

«Как себя чувствуешь сегодня?» — пользователь выбирает из кнопок: Отлично / Хорошо / Нормально / Плохо / Болел. Опционально добавляет теги: «низкая энергия», «плохой сон».

## 5. Подробный функционал

### 5.1 Модуль питания

**Ввод через Telegram:**
- Свободный текст на русском: «съел 200г куриной грудки и 150г риса»
- Бот распознаёт продукты через базу данных + AI-интерпретацию
- При нераспознанном продукте бот уточняет: «Не нашёл "паста спагетти с брынзой" как одно блюдо. Это готовый рецепт или ввести по частям: спагетти + брынза?»
- Поддержка составных блюд: сохранённые рецепты (борщ, запеканка) — одна команда раскрывает все ингредиенты

**Продукты питания:**
- Рис, гречка, куриная грудка, лосось, паста, творог, сметана, говядина, тунец консервированный, овощи (помидоры, огурцы, перец, капуста, морковь), оливки, банан, квашеная капуста и другие — всё хранится в личной базе с ценами
- Ввод граммовки точный (с весами)
- Для ресторана: заранее созданные заготовки «пицца маргарита ~350г», «бриошь с кремом ~180г», «кофе американо»

**Напитки:**
- Вода (цель на день, прогресс)
- Кофе (нейтральный учёт)
- Пиво — особая метка «нежелательное». При добавлении бот фиксирует без осуждения, но в еженедельном отчёте показывает частоту и напоминает о цели

**Добавки и витамины:**
- Отдельная категория, не влияет на БЖУ расчёт (пока не принимаются, но структура готова)

### 5.2 Рецепты и составные блюда

- Создание рецепта: название, список ингредиентов с граммовкой, количество порций
- При добавлении рецепта в дневник: указывается количество порций → автоматический пересчёт
- Пример: «Борщ (6 порций): свёкла 300г, капуста 200г, картошка 250г, говядина 400г, морковь 100г, лук 80г» → при вводе «съел 1 порцию борща» система рассчитывает БЖУ на 1/6 рецепта

### 5.3 Цели и нормы

- Ежедневные цели по Б/Ж/У/клетчатке (граммы)
- Калорийная цель (рассчитывается автоматически из БЖУ или задаётся вручную)
- Цель по воде (мл)
- Коррекция дневной нормы калорий в дни тренировок (+ расход от тренировки)
- Бюджетная цель на питание (€/неделю или €/месяц)

### 5.4 Модуль тренировок

**Типы тренировок:**
- Домашняя (гиря, собственный вес, турник, отжимания, приседания)
- Бег / велосипед / плавание (если появятся)

**Данные по тренировке:**
- Тип тренировки
- Упражнения: название, подходы × повторения × вес (кг)
- Длительность (минуты)
- Субъективная сложность (1-10)
- Рассчитанный расход калорий (на основе типа, длительности и веса тела)
- Заметки

**Шаблоны:**
- Предустановленные шаблоны (например «Базовая домашняя»: турник + отжимания + приседания)
- Создание собственных шаблонов
- Быстрый старт из шаблона с возможностью корректировки перед сохранением

**Прогресс:**
- История рабочих весов по каждому упражнению
- График повторений/веса во времени
- Персональные рекорды (ПР) с автоматическим фиксированием

### 5.5 Модуль самочувствия

**Ежедневная оценка (через бота или веб):**
- Общее состояние: Отлично / Хорошо / Нормально / Плохо / Болел
- Опциональные теги: уровень энергии (1-5), качество сна (1-5), стресс (1-5), настроение (1-5)
- Флаги симптомов: боль в спине, боль в суставах, усталость, головная боль

**Частота:**
- 1-2 раза в день по желанию
- Утреннее напоминание (опционально)
- Вечернее напоминание (опционально)

### 5.6 Учёт веса тела

- Утренний вес (после туалета, до еды) — рекомендуемое время
- График в динамике
- Скользящее среднее за 7 дней (убирает дневные колебания)
- Связь с питанием: корреляция веса с потреблением калорий, углеводов

## 6. Архитектура Telegram-бота

### 6.1 Общая структура

```
/start          — регистрация, выбор языка, первичная настройка целей
/goals          — просмотр и редактирование дневных целей
/today          — текущий прогресс за день
/log [текст]    — добавить приём пищи
/weight [число] — записать вес
/train          — начать тренировку (шаблон или новая)
/feel           — оценить самочувствие
/receipt        — загрузить фото чека
/stats          — быстрая недельная статистика
/help           — справка
```

### 6.2 Умный парсинг еды

**Алгоритм обработки сообщения:**

1. Пользователь пишет в свободной форме (без команды — бот понимает контекст)
2. AI-парсер выделяет продукты и количества
3. Поиск по личной базе продуктов пользователя (приоритет)
4. Поиск по общей базе продуктов
5. Если продукт не найден → уточняющий вопрос с кнопками-вариантами
6. Расчёт БЖУ и калорий
7. Ответ с итогами + обновлённый прогресс на день

**Обработка неточных запросов:**
- «Поел в пиццерии» → бот: «Какую пиццу? У тебя есть сохранённый вариант "Пицца маргарита ~350г". Использовать?»
- «Выпил кофе» → добавляется как кофе без молока (250 мл, 2 ккал), если не уточнено
- «Съел что-то жирное» → бот просит уточнить. Варианты: «Пропустить», «Ввести вручную», «Выбрать из списка»

### 6.3 Сценарий для нетехнического пользователя

**Простой режим (по умолчанию):**

Бот присылает чёткие кнопки вместо текстовых команд:
- 🍽 Добавить еду
- 💪 Записать тренировку
- ⚖️ Записать вес
- 😊 Как себя чувствую
- 📊 Мой прогресс сегодня

После нажатия «Добавить еду» — пошаговый диалог:
1. «Что ты ел? Напиши или выбери из частых:» [кнопки: Куриная грудка | Рис | Гречка | Яйца | Другое]
2. «Сколько граммов?» — пользователь вводит число
3. Бот подтверждает: «Добавлено: 200г куриной грудки. Белок: +48г. Осталось до цели: Б 52г / Ж 100г / У 200г»

### 6.4 Напоминания

- Утреннее (настраиваемое время): «Доброе утро! Вчерашние итоги + цели на сегодня»
- Обеденное (опционально): «Не забудь записать обед!»
- Вечернее (опционально): «Оцени самочувствие за день»
- Напоминание о тренировке (если запланирована)
- Напоминание о воде (если выбрана опция)

### 6.5 Редактирование записей

Команда `/edit` или кнопка в ответном сообщении позволяет:
- Изменить последний приём пищи
- Удалить запись
- Скорректировать граммовку

## 7. Архитектура Web-приложения

### 7.1 Структура страниц

```
/                  — Dashboard (главный экран)
/diary             — Дневник питания
/nutrition         — Детальная аналитика нутриентов
/cost              — Стоимость питания и анализ бюджета
/workouts          — Журнал тренировок
/workouts/[id]     — Детали тренировки
/recipes           — Рецепты и составные блюда
/wellbeing         — Дневник самочувствия
/weight            — График веса
/products          — База продуктов (личная)
/receipts          — Загруженные чеки
/analytics         — Причинно-следственные связи
/settings          — Настройки: цели, профиль, уведомления
/family            — Семейный режим
```

### 7.2 Dashboard (главный экран)

**Верхняя часть — «Сегодня»:**
- Прогресс-бары: Белки / Жиры / Углеводы / Клетчатка (факт / цель)
- Кольцо калорий: потреблено / сожжено на тренировке / дневная норма
- Вес сегодня (если внесён)
- Вода: стаканы / цель

**Средняя часть — «Быстрые действия»:**
- Кнопка «Добавить приём пищи» (открывает форму)
- Кнопка «Записать тренировку»
- Кнопка «Взвеситься»

**Нижняя часть — «Инсайты недели»:**
- 2-3 автоматически сгенерированных наблюдения
- Последняя тренировка (сколько дней назад)
- Бюджет на еду: потрачено / цель за неделю

### 7.3 Дневник питания

- Приёмы пищи по времени (хронологический список)
- Группировка: Завтрак / Обед / Ужин / Перекус (по времени ввода)
- Для каждого приёма: продукты, граммы, БЖУ, калории, стоимость
- Итог дня: суммарные БЖУ, калории, стоимость
- Навигация по дням

### 7.4 Страница стоимости питания

- Бюджет за день / неделю / месяц
- Топ-5 самых дорогих продуктов по расходу
- Топ-5 самых выгодных по стоимости белка (€/100г белка)
- Сравнение магазинов: где покупался продукт дешевле
- График расходов на еду по неделям

## 9. Система хранения и обработки чеков

### 9.1 Архитектура обработки чека

**Шаг 1 — Загрузка**
Пользователь отправляет фото чека в Telegram-бот или загружает через веб.
Изображение сохраняется в хранилище (Cloudinary / локальный сервер).

**Шаг 2 — OCR-распознавание**
- Используется Google Vision API (или Tesseract для бесплатного варианта)
- Извлекается текст со структурой: название товара / количество / цена

**Шаг 3 — Нормализация**
Чеки итальянских супермаркетов (Conad, Lidl, Pan) имеют стандартные форматы. Создаётся парсер под каждый формат:
- «PETTO POLLO KG» → «куриная грудка»
- «SALMONE NORV. AFU» → «лосось»
- Нераспознанные → очередь на ручное подтверждение

**Шаг 4 — Фильтрация**
Автоматически пропускаются позиции с категориями: бытовая химия, косметика, алкоголь (не еда). Правила фильтрации настраиваются.

**Шаг 5 — Сопоставление с базой продуктов**
- Fuzzy matching между названием в чеке и именами продуктов в базе
- Если совпадение >85% → автоматически привязывается
- Если 60-85% → предлагается пользователю подтвердить
- Если <60% → пользователь создаёт новый продукт или пропускает

**Шаг 6 — Обновление цен**
- Рассчитывается цена за кг (или за единицу)
- Обновляется `priceHistory` в продукте с указанием магазина и даты
- Текущая цена (`currentPrice`) = последняя известная цена

**Шаг 7 — Подтверждение пользователем**
Бот присылает список распознанных продуктов с ценами:
```
Чек Conad от 31.05.2025:

✅ Куриная грудка — 1.2 кг — 7.08€ (5.90€/кг)
✅ Рис — 1 кг — 1.29€
❓ MELANZ. LUNGHE GRILL 300G — не распознано [Подтвердить | Пропустить]

Итого продуктов питания: 8.37€
```

### 9.2 Учёт остатков (опционально, v2.0)

- После распознавания чека система знает: куплено X грамм продукта по цене Y
- При каждом добавлении в дневник: остаток уменьшается
- Когда остаток < 20% → уведомление «Куриная грудка заканчивается»

## 10. Система учёта стоимости нутриентов

### 10.1 Базовый расчёт

Для каждого приёма пищи и каждого продукта рассчитывается:

```
stCostPerProtein = (portionCostEur / proteinGrams) × 100
// €/100г белка

stCostPerCarbs = (portionCostEur / carbsGrams) × 100
stCostPerFat = (portionCostEur / fatGrams) × 100
stCostPerFiber = (portionCostEur / fiberGrams) × 100
```

**Пример:**
| Продукт | Цена 200г | Белок | €/100г Б |
|---------|-----------|-------|-----------|
| Тунец консерв. | 2.40€ | 36г | 6.67€ |
| Куриная грудка | 1.18€ | 47г | 2.51€ |
| Творог 9% | 0.60€ | 34г | 1.76€ |
| Яйца (3 шт.) | 0.45€ | 19г | 2.37€ |

### 10.2 Рейтинг эффективности продуктов

**По белку (€/100г белка) — чем меньше, тем лучше:**
1. 🥇 Творог — 1.76€
2. 🥈 Яйца — 2.37€
3. 🥉 Куриная грудка — 2.51€
4. Тунец — 6.67€
5. Лосось — 9.20€

**По клетчатке (€/10г клетчатки):**
1. 🥇 Гречка — 0.18€
2. 🥈 Капуста свежая — 0.22€
3. 🥉 Рис — 0.31€

### 10.3 Сравнение магазинов

```
Куриная грудка:
  Lidl     — 4.80€/кг  ✅ (последняя покупка 14 дней назад)
  Conad    — 5.90€/кг
  Pan      — 6.20€/кг

Потенциальная экономия при покупке в Lidl: 1.10€/кг
```

### 10.4 Оптимизатор недельного меню (v2.0)

На основе целей по БЖУ и бюджету система предлагает оптимальный набор продуктов на неделю, минимизирующий стоимость при выполнении нутриентных целей.

## 11. Система аналитики и рекомендаций

### 11.1 Простые наблюдения (MVP)

Алгоритм «паттернов» на основе накопленных данных (минимум 14 дней):

**Питание → Самочувствие:**
- Если в ≥70% случаев вечернее употребление продукта X коррелирует с плохим сном → показать пользователю
- Пример вывода: «Замечено: в 8 из 10 дней, когда ты ел жареную еду после 20:00, качество сна было оценено ниже 3/5»

**Тренировки → Боль:**
- Если после упражнения Y в ≥60% случаев следует симптом «боль в пояснице» на следующий день → предупреждение
- Пример: «После становой тяги (5 из 8 тренировок) на следующий день ты отмечал боль в пояснице»

**Питание → Вес:**
- Корреляция калорийного дефицита/профицита с изменением веса
- Влияние углеводов на утренний вес (ретенция воды)

**Активность → Самочувствие:**
- «В дни после тренировки твой уровень энергии на 0.8 балла выше среднего»

### 11.2 Недельные инсайты (автогенерация)

Каждое воскресенье формируется краткий отчёт:
- Выполнение целей по БЖУ за неделю (%)
- Тренировки: было / запланировано
- Вес: динамика за неделю
- Бюджет: потрачено / план
- 2-3 наблюдения из системы паттернов

### 11.3 AI-рекомендации по питанию (с Claude API)

При дефиците нутриентов система генерирует персональные рекомендации:

```
Запрос к AI: «Пользователь выполнил план по белку на 62% за последние 7 дней. 
Его обычные продукты: [список]. Бюджет 80€/неделю. 
Дай 2-3 конкретных рекомендации как добраться до 100г белка/день»

Ответ: «На основе твоих продуктов: добавь 2 яйца к завтраку (+13г Б, +0.30€), 
замени вечерний рис на гречку (+2г Б на 100г) и добавь 150г творога как перекус (+24г Б, 0.60€)»
```

## 13. Семейный режим

### 13.1 Концепция

Два пользователя (на старте) связаны в «семейную группу». Каждый видит:
- Свои детальные данные (личный дневник)
- Сводку прогресса другого (без деталей, только итоги дня)
- Общий семейный дашборд

### 13.2 Общий дашборд семьи

```
Семейный дашборд — сегодня:

Алексей:  Б ████░ 78%  Тренировка: ✅  Самочувствие: 😊
Мария:    Б ██░░░ 45%  Тренировка: —   Самочувствие: 😐

Семейный бюджет на еду: 47€ из 150€/неделю ████░░░░░░ 31%
```

### 13.3 Функции поддержки

- Взаимные уведомления: «Алексей выполнил план по белку 5 дней подряд! 💪»
- Напоминания: если один из членов не внёс данные за день → партнёр видит пустой статус
- Совместный бюджет на еду: общая копилка и отслеживание расходов
- Общие рецепты: созданные одним участником доступны обоим

### 13.4 Приватность

- Детальный дневник питания виден только владельцу
- Партнёр видит только агрегированные данные: выполнение целей (%), тренировка да/нет, самочувствие (эмоджи)
- Настройка уровня открытости в параметрах профиля

## 14. Соревновательный режим

### 14.1 Концепция (без очков и рейтингов)

Не соревнование «кто лучше», а **взаимная видимость** и **мотивационная поддержка**. Идея: когда видишь, что партнёр тренировался, хочется не отставать — и наоборот.

### 14.2 Механики

**«Серии» (streak):**
- Личные серии: «5 дней подряд выполнен план по белку»
- Видимость серии партнёра — мотивирует не прерывать свою

**Взаимные уведомления:**
- «Мария сегодня уже потренировалась» → мотивация для Алексея
- «Алексей не вносил данные сегодня, напомни ему?» → забота, не давление

**Совместные недельные цели:**
- «На этой неделе оба тренируемся минимум 3 раза»
- Статус: Алексей ✅✅☐ / Мария ✅☐☐

**Отсутствие осуждения:**
- Нет штрафов за пропуски
- Нет публичного позора
- Только позитивные подкрепления

## 16. Возможные риски

### 16.1 Технические риски

| Риск | Вероятность | Решение |
|------|------------|---------|

### 16.2 Продуктовые риски

| Риск | Вероятность | Решение |
|------|------------|---------|
| Второй пользователь не вносит данные регулярно | Высокая | Максимально упростить ввод; кнопочный режим; только самое важное |

## 17. Что можно улучшить в исходной идее

### Критика и улучшения

**1. Система целей слишком статична**
Исходная идея: фиксированные цели Б100/Ж100/У200 каждый день.
Проблема: в день тренировки потребность в белке выше; в день отдыха — калорийность можно снизить.
Решение: динамические цели — базовые + коррекция на тренировку (расход калорий → разрешённый профицит калорий и белка).

**2. Нет контекста «почему эта цель»**
Цифры БЖУ без понимания — просто числа. Первый раз при настройке бот должен объяснить: «100г белка — это примерно 1.2г/кг твоего веса. Это стандартная норма для поддержания мышц при домашних тренировках».

**3. Учёт чеков может стать барьером**
Загрузка фото + подтверждение позиций — это работа. Для MVP достаточно ручного ввода цен. Чеки — фича v2.0.

**4. Пиво как «нежелательное» — хорошая идея, но требует деликатности**
Не нужно морализировать. Просто статистика: «На этой неделе: пиво 2 раза». Без комментариев. Пользователь сам сделает выводы.

**5. Отсутствие идеи «быстрого повтора»**
Люди едят примерно одинаково изо дня в день. 80% дней — одни и те же 10-15 блюд. Нужна функция «Повторить завтрак как вчера» — один клик, и всё готово.

## 18. Какие функции отсутствуют в исходной идее

### Добавленные ценные функции:

1. **«Повторить приём пищи»** — самая используемая функция в подобных приложениях. Без неё усталость от ввода наступает быстро.

2. **Скользящее среднее веса** — убирает ежедневные колебания (соль, вода) и показывает реальную тенденцию.

3. **Прогресс-рекорды в тренировках** — «Новый рекорд! Подтягивания 12 повторений» — один из сильнейших мотиваторов.

4. **Детальный учёт кофе** — кофе у тебя ежедневный ритуал. Учёт количества чашек + связь с самочувствием (много кофе → стресс/сон) может дать интересные инсайты.

5. **«Умный» список покупок** — на основе рецептов недели и остатков система формирует список того, что нужно купить. Экономит время и деньги.

6. **Дефицит/профицит калорий** — простой визуальный показатель: сегодня +200 ккал или -150 ккал к норме. Пользователь видит тренд.

7. **Шаблонный план питания на неделю** — не жёсткий, а мягкий: «На этой неделе попробуй вот это меню». Снимает проблему «постоянно думать, что приготовить» — ту самую боль, которую ты описал.

## 19. Сильнейшие конкурентные преимущества

### ТОП-5 уникальных преимуществ NutriTrack:

**1. Стоимость нутриентов (€/г белка)**
Ни одно существующее приложение не считает, сколько стоит каждый грамм белка в конкретном продукте. Это уникальная функция для людей, которые следят и за питанием, и за бюджетом одновременно.

**2. Сравнение цен продукта по магазинам**
«Куриная грудка в Lidl на 18% дешевле, чем в Conad» — это ценная информация, которую MyFitnessPal никогда не даст.

**3. Единая система: еда + тренировки + самочувствие + деньги**
Конкуренты либо специализированы (только еда / только тренировки), либо слишком дорогие (Whoop = $300/год). NutriTrack — всё в одном, бесплатно для личного использования.

**4. Telegram как точка входа**
Не нужно устанавливать приложение. Telegram уже открыт 24/7. Это снижает барьер ввода данных до абсолютного минимума.

**5. Семейный режим с совместным бюджетом**
Большинство людей покупают продукты на семью. Учёт семейного бюджета на питание — отсутствующая функция во всех конкурентах.

## 20. Финальная улучшенная концепция продукта

### NutriTrack — «Умный советник по питанию, здоровью и бюджету»

**Суть продукта в одном абзаце:**
NutriTrack — это персональный система контроля, которая работает через Telegram (ввод данных без барьеров) и веб-приложение (аналитика и планирование). Она объединяет учёт питания с точным расчётом стоимости нутриентов, журнал тренировок с отслеживанием прогресса, дневник самочувствия и умную аналитику, которая находит причины и следствия между всеми этими данными. Для семьи из двух человек это также инструмент совместного планирования бюджета на питание и взаимной мотивации.

**Главный вопрос, на который отвечает продукт:**
*«Правильно ли я питаюсь, укладываюсь ли в бюджет, и что именно влияет на моё самочувствие и прогресс?»*

**Технический стек:**
- Backend: Node.js + Express
- Database: MongoDB Atlas
- Bot: Telegraf (Telegram Bot Framework)
- Frontend: React + Recharts (графики)
- AI: Claude API (парсинг еды, рекомендации)
- OCR: Tesseract.js (бесплатно) → Google Vision API (v2.0)
- Hosting: Railway / Render (бесплатные тиры для MVP)
- Storage: Cloudinary (бесплатный тир для фото чеков)

**Принципы разработки:**
1. Сначала данные — потом красота. MVP: всё работает, выглядит просто.
2. Второй пользователь — первоклассный гражданин. Каждая функция тестируется на нетехническом пользователе.
3. Один ввод → максимум информации. Написал «съел 200г курицы» → получил БЖУ + калории + стоимость + прогресс за день.
4. Инсайты только тогда, когда данных достаточно. Нет данных → нет инсайтов. Лучше молчать, чем ошибаться.
5. Деньги — равноправная метрика здоровья. Стоимость питания не второстепенна — она на главном дашборде.
